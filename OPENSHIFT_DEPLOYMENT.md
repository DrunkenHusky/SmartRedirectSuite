### Wie Environment Variables funktionieren

Die Anwendung verwendet `dotenv/config` (siehe `server/index.ts` Zeile 1), was bedeutet:

1. **Lokale Entwicklung**: Die Anwendung liest `.env` Dateien automatisch
2. **OpenShift Deployment**: Environment Variables aus ConfigMaps und Secrets √ºberschreiben automatisch alle `.env` Werte
3. **Priorit√§t**: OpenShift Environment Variables > .env Datei > Default-Werte im Code

**Praktisches Beispiel:**

// Verhalten:
// - Lokal: Liest aus .env Datei
// - OpenShift: Verwendet ConfigMap/Secret-Werte
// - Fallback: 'default-value' wenn nichts gesetzt

Die Anwendung **erkennt automatisch** die Umgebung und verwendet die korrekten Werte ohne zus√§tzliche Konfiguration.
